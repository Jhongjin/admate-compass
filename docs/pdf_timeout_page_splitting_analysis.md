# 특정 PDF 파일 타임아웃 문제 및 페이지 분할 계획 검토

**작성일**: 2025-11-06  
**문제 파일**: `2025_Introduction_to_Meta_Meta_기본소개서_externa.pdf`

---

## 🔍 문제 분석

### 현재 상황
- **파일**: `2025_Introduction_to_Meta_Meta_기본소개서_externa.pdf`
- **에러**: `큐 처리 시간 초과 (10분)`
- **시도 횟수**: `0/3` (재시도 실패)
- **처리 시간**: 약 10분 (타임아웃 직전까지 실행)

### 다른 PDF 파일 처리 상태
- 다른 PDF 파일들은 정상 처리됨 (1-2개 청크)
- 성공한 파일들은 상대적으로 작은 크기

---

## 💡 페이지 분할 계획의 해결 가능성

### ✅ **매우 높은 해결 가능성**

페이지/섹션 분할 계획(`docs/long_term_plan_page_section_splitting.md`)은 이 문제를 **근본적으로 해결할 수 있는 매우 유력한 방안**입니다.

### 해결 메커니즘

#### 1. **타임아웃 문제 해결**
- **현재**: 전체 PDF를 한 번에 처리 → 10분 초과
- **분할 후**: 페이지/섹션 단위로 분할하여 각각 처리
  - 각 분할은 1-2분 내 처리 가능
  - 타임아웃(10분) 내에서 완료 가능

#### 2. **메모리 사용량 감소**
- **현재**: 전체 PDF를 메모리에 로드 (Vercel 제한: 1024MB)
- **분할 후**: 각 분할만 메모리에 로드
  - 메모리 사용량 크게 감소
  - OOM(Out of Memory) 에러 방지

#### 3. **병렬 처리 가능**
- 각 분할을 독립적으로 처리
- 여러 분할을 동시에 처리하여 전체 시간 단축

#### 4. **부분 실패 허용**
- **현재**: 한 부분 실패 시 전체 실패
- **분할 후**: 일부 분할 실패해도 나머지 처리 가능
  - 부분적 성공 허용
  - 실패한 분할만 재시도

## 📋 구현 계획 요약

### Phase 1: 인프라 준비 (1주)
- `CHUNK_PROCESS` job 타입 추가
- `document_splits` 테이블 생성
- 분할 상태 추적 (`split_status` JSONB)

### Phase 2: PDF 구조 분석 (2주)
- 페이지별 텍스트 추출
- 섹션/챕터 감지
- 분할 전략 결정

### Phase 3: 분할 처리 로직 (2주)
- 텍스트 분할 서비스 구현
- 큐에 분할 작업 등록
- 개별 분할 처리

### Phase 4: UI 개선 (1주)
- 진행 상황 표시
- 분할별 상태 표시

### Phase 5: 테스트 및 최적화 (1주)
- 다양한 PDF 테스트
- 성능 최적화

**총 예상 기간**: 7주

## 🎯 즉시 적용 가능한 임시 해결책

페이지 분할 계획 구현 전까지 다음을 적용할 수 있습니다:

### 1. 파일 크기 제한 강화
- 현재: 15MB
- 권장: **10MB** (더 안정적인 처리)

### 2. PDF 페이지 수 제한
- `pdf-parse`는 페이지 수 제한 직접 지원 안 함
- 대신: 텍스트 길이 기반 제한
- 권장: **500KB 텍스트** 이상 시 경고

### 3. 재시도 로직 개선
- 타임아웃 발생 시 자동 재시도 (현재 구현됨)
- 재시도 간격 조정

## 📊 예상 효과

### 페이지 분할 적용 후:
- **타임아웃 실패율**: 100% → **0%** (예상)
- **처리 성공률**: 0% → **90%+** (예상)
- **처리 시간**: 10분 초과 → **2-3분** (분할 병렬 처리 시)

### 특히 큰 파일:
- **15MB+ PDF**: 현재 실패 → 분할 후 성공 가능
- **100페이지+ PDF**: 현재 실패 → 분할 후 성공 가능

## 🚀 결론

**페이지 분할 계획은 특정 PDF 파일 타임아웃 문제를 해결하는 최적의 방법입니다.**

단기적으로는 파일 크기 제한과 재시도 로직으로 완화할 수 있지만, 근본적인 해결을 위해서는 페이지 분할 계획 구현이 필요합니다.

**권장 사항**:
1. **단기 (즉시)**: 파일 크기 제한 10MB로 강화
2. **중기 (1-2주)**: 페이지 분할 계획 Phase 1-2 구현
3. **장기 (7주)**: 전체 페이지 분할 시스템 구현
