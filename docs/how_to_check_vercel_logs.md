# Vercel 로그 확인 방법 (단계별 가이드)

## 📍 Vercel 로그 위치

### 1단계: Vercel 대시보드 접속
1. 브라우저에서 [https://vercel.com/dashboard](https://vercel.com/dashboard) 접속
2. 로그인 후 프로젝트 선택

### 2단계: Logs 탭 열기
1. 프로젝트 대시보드에서 상단 메뉴 확인
2. **"Logs"** 탭 클릭
   - 또는 **"Functions"** → 특정 함수 선택 → **"Logs"** 탭

### 3단계: 로그 필터링
1. **Function** 드롭다운에서 `/api/jobs/consume` 선택
2. **Time Range** 설정 (최근 10분 또는 1시간)
3. 검색창에 키워드 입력 (예: "큐 처리 시작", "DOCX 파싱")

## 🔍 확인해야 할 로그

### 큐 워커 실행 여부
**검색 키워드**: `🚀 큐 처리 시작`

**예상 로그**:
```
🚀 큐 처리 시작: 2025-01-XX...
🔍 큐에서 작업 조회 중...
📋 작업 조회 완료: 50ms
✅ 작업 선택됨: { jobType: 'DOCX_PARSE', documentId: 'doc_...' }
```

**문제**: 이 로그가 없으면 → 큐 워커가 실행되지 않고 있음

---

### Storage 다운로드 시간
**검색 키워드**: `Storage 다운로드`

**예상 로그**:
```
📥 Storage 다운로드 시작: documents/doc_.../...
✅ Storage 다운로드 완료: 0.5MB (200ms)
```

**정상**: 200-500ms
**느림**: 5000ms 이상 → Storage 연결 문제

---

### DOCX 파싱 시간
**검색 키워드**: `DOCX 파싱`

**예상 로그**:
```
📄 DOCX 파싱 시작: 0.5MB
✅ DOCX 파싱 완료: 50KB (300ms)
```

**정상**: 100-500ms (작은 파일)
**느림**: 5000ms 이상 → mammoth 라이브러리 문제

---

### 청킹 시간
**검색 키워드**: `문서 청킹`

**예상 로그**:
```
📄 문서 청킹 시작...
✅ 문서 청킹 완료: { chunkCount: 10, time: '500ms' }
```

**정상**: 100-1000ms
**느림**: 5000ms 이상 → 청킹 로직 문제

---

### 임베딩 생성 시간
**검색 키워드**: `임베딩 생성`

**예상 로그**:
```
🔮 임베딩 생성 시작... { chunkCount: 10 }
✅ 임베딩 생성 완료: { chunkCount: 10, time: '1000ms', avgTimePerChunk: '100ms' }
```

**정상**: 100ms/청크
**느림**: 1000ms/청크 이상 → 임베딩 생성 로직 문제

---

### 저장 시간
**검색 키워드**: `청크 저장`

**예상 로그**:
```
💾 작은 파일 청크 저장 시작: 10개 청크 (한 번에 저장)
✅ 청크 저장 완료: 10개 청크 (800ms)
```

**정상**: 50-100ms/청크
**느림**: 500ms/청크 이상 → 데이터베이스 연결 문제

---

## 🚨 즉시 확인 사항

### 1. 큐 워커가 실행되고 있는지
- [ ] `🚀 큐 처리 시작` 로그가 최근 1분 내에 있는가?
- [ ] 없으면 → 수동 실행 필요 (아래 참고)

### 2. 어느 단계가 느린지
- [ ] Storage 다운로드: `✅ Storage 다운로드 완료`의 ms 값 확인
- [ ] DOCX 파싱: `✅ DOCX 파싱 완료`의 ms 값 확인
- [ ] 청킹: `✅ 문서 청킹 완료`의 time 값 확인
- [ ] 임베딩: `✅ 임베딩 생성 완료`의 time 값 확인
- [ ] 저장: `✅ 청크 저장 완료`의 time 값 확인

### 3. 에러가 있는지
- [ ] `❌`로 시작하는 로그 확인
- [ ] 타임아웃 에러 확인
- [ ] 파싱 에러 확인

---

## 🔧 수동 실행 방법

### 방법 1: 관리자 페이지 (가장 쉬움)
1. `/admin/docs` 페이지 접속
2. 좌측 하단 **"처리 큐 (요약)"** 패널
3. **"1건 처리"** 버튼 클릭
4. Vercel Logs에서 실시간 로그 확인

### 방법 2: API 직접 호출
브라우저 개발자 도구 Console에서:
```javascript
fetch('/api/jobs/consume', { method: 'POST' })
  .then(r => r.json())
  .then(console.log)
  .catch(console.error);
```

```
🚀 큐 처리 시작: 2025-01-XX...
📋 작업 조회 완료: 50ms
✅ 작업 선택됨: { jobType: 'DOCX_PARSE' }
📥 Storage 다운로드 시작: documents/...
✅ Storage 다운로드 완료: 0.5MB (200ms)
📄 DOCX 파싱 시작: 0.5MB
✅ DOCX 파싱 완료: 50KB (300ms)
📊 텍스트 추출 검증: { rawLength: 50000, cleanedLength: 49000 }
📄 문서 청킹 시작...
✅ 문서 청킹 완료: { chunkCount: 10, time: '500ms' }
🔮 임베딩 생성 시작... { chunkCount: 10 }
✅ 임베딩 생성 완료: { chunkCount: 10, time: '1000ms', avgTimePerChunk: '100ms' }
💾 작은 파일 청크 저장 시작: 10개 청크
✅ 청크 저장 완료: 10개 청크 (800ms)
```

**총 시간**: 약 2.8초 (정상)

### 느린 처리 (문제 발생)
```
🚀 큐 처리 시작: 2025-01-XX...
📥 Storage 다운로드 시작: documents/...
✅ Storage 다운로드 완료: 0.5MB (5000ms)  ← 여기서 느림!
...
```

또는

```
🚀 큐 처리 시작: 2025-01-XX...
📥 Storage 다운로드 시작: documents/...
(로그 없음 - 10분 후 타임아웃)  ← 여기서 멈춤!
```
