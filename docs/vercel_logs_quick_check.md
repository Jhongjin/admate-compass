# Vercel 로그 빠른 확인 가이드

## 📍 로그 위치 (단계별)

### 1단계: Vercel 대시보드
1. https://vercel.com/dashboard 접속
2. 프로젝트 선택

### 2단계: Logs 탭
1. 상단 메뉴에서 **"Logs"** 클릭
2. 또는 **"Functions"** → `/api/jobs/consume` 선택 → **"Logs"** 탭

### 3단계: 필터링
1. **Function**: `/api/jobs/consume` 선택
2. **Time Range**: 최근 10분 선택
3. **Search**: `큐 처리 시작` 입력

---

## 🔍 확인할 로그 키워드

### 큐 워커 실행 여부
```
🚀 큐 처리 시작
```
**이 로그가 없으면 → 큐 워커가 실행되지 않음**

### 각 단계별 시간
```
📥 Storage 다운로드 완료: XXMB (XXms)
📄 DOCX 파싱 완료: XXKB (XXms)
✅ 문서 청킹 완료: { chunkCount: X, time: 'XXXms' }
✅ 임베딩 생성 완료: { time: 'XXXms' }
✅ 청크 저장 완료: { time: 'XXXms' }
```

## 🚀 즉시 테스트 방법

### 방법 1: 관리자 페이지에서 수동 실행
1. `/admin/docs` 페이지 접속
2. 좌측 하단 **"처리 큐 (요약)"** 패널
3. **"1건 처리"** 버튼 클릭
4. **동시에 Vercel Logs 탭 열어두기** (실시간 확인)

### 방법 2: 브라우저 개발자 도구
1. `/admin/docs` 페이지 열기
2. **F12** (개발자 도구 열기)
3. **Network** 탭 열기
4. **"1건 처리"** 버튼 클릭
5. `/api/jobs/consume` 요청 확인
6. **Response** 탭에서 결과 확인

## ❌ 문제 발생 시 로그

### 큐 워커가 실행되지 않는 경우
- 로그에 `🚀 큐 처리 시작`이 없음
- 해결: 관리자 페이지에서 "1건 처리" 버튼 클릭

### Storage 다운로드가 느린 경우
```
✅ Storage 다운로드 완료: 0.5MB (5000ms)  ← 5초 이상
```
- 문제: Supabase Storage 연결 지연
- 해결: 네트워크 확인, Storage 설정 확인

### DOCX 파싱이 느린 경우
```
✅ DOCX 파싱 완료: 50KB (5000ms)  ← 5초 이상
```
- 문제: mammoth 라이브러리 성능 문제
- 해결: 파일 크기 확인, 대체 라이브러리 검토

### 임베딩 생성이 느린 경우
```
✅ 임베딩 생성 완료: { time: '50000ms' }  ← 50초 이상
```
- 문제: 임베딩 생성 로직 문제
- 해결: 배치 크기 조정, 로직 최적화

### 저장이 느린 경우
```
✅ 청크 저장 완료: { time: '50000ms' }  ← 50초 이상
```
- 문제: 데이터베이스 연결 지연
- 해결: DB 연결 확인, 인덱스 확인
