# 큐 처리 실패 종합 원인 분석 및 해결 방안

**작성일**: 2025-01-31  
**문제**: 타임아웃/용량 외 다른 전반적인 원인 분석 필요

---

## 🔍 발견된 근본 원인

### 1. ❌ Promise.race()로 인한 실제 처리 중단 실패

**현재 코드**:
```typescript
const processPromise = ragProcessor.processDocument(docData, true);
const timeoutPromise = new Promise((_, reject) => {
  setTimeout(() => reject(new Error('타임아웃')), MAX_PROCESS_TIME);
});

processResult = await Promise.race([processPromise, timeoutPromise]);
```

**문제점**:
1. 타임아웃 발생 시 `processPromise`는 계속 실행됨
2. 메모리 누수 가능성
3. 다음 재시도 시 이미 실행 중인 작업과 충돌 가능

**해결책**: AbortController를 사용하여 실제 처리 중단

---

### 2. ⚠️ Vercel Cron Job 실행 빈도 문제

**현재 설정**: `*/1 * * * *` (1분마다 실행)

**문제점**:
- 하나의 작업이 10분 걸리면 다음 Cron 실행이 대기
- 동시에 여러 작업을 처리할 수 없음
- 재시도 작업도 같은 제약 영향

**해결책**: Cron 실행 빈도 증가 또는 수동 처리 지원

---

### 3. 🚨 PDF 파싱 성능 문제

**문제점**:
- `pdf-parse`는 전체 PDF를 메모리에 로드
- 큰 PDF는 파싱 시간이 비선형 증가
- 13MB PDF → 2-5분 소요

**해결책**: 페이지 수 제한 또는 스트리밍 파싱

---

### 4. 💾 메모리 제한 문제

**Vercel Pro 제약**:
- 메모리: 1024MB
- 큰 파일 처리 시 메모리 부족 가능

**해결책**: 청크 단위 처리 또는 메모리 사용량 모니터링

---

### 5. ⏱️ 순차 처리로 인한 시간 누적

**현재 파이프라인**:
```
다운로드 → 파싱 → 정규화 → 청킹 → 임베딩 → 저장
```

**문제점**: 각 단계가 순차적으로 실행되어 시간 누적

**해결책**: 가능한 단계는 병렬 처리

---
