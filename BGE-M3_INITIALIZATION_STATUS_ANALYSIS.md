00:57:34.100 - 하트비트 2회 (10.1초 경과) ✅

**✅ 정상 진행 중:**

**예상 완료 시간:**

## 🔍 진행 중인지 무한 대기 중인지 확인 방법

### 방법 1: 하트비트 로그 확인

**정상 진행 중:**
- 5초마다 하트비트 로그 발생
- 경과 시간이 계속 증가
- 하트비트 카운트 증가

**무한 대기 중:**
- 하트비트 로그가 멈춤
- 경과 시간이 더 이상 증가하지 않음
- 마지막 로그 이후 아무 로그도 없음

**현재 상태:**
- ✅ 하트비트 정상 작동 (5초마다)
- ✅ 경과 시간 증가 (5.0초 → 10.1초)
- ✅ 진행 중으로 판단

### 방법 2: Vercel 로그 실시간 확인

1. **Vercel Dashboard 접속**
2. **프로젝트 선택**
3. **Functions 탭** → `/api/jobs/consume` 함수 선택
4. **Logs 탭**에서 실시간 로그 확인

**확인할 로그 패턴:**

**정상 진행 중:**
```

⏳ BGE-M3 모델 초기화 진행 중... (경과: 25.0초, 하트비트: 5회)
...
✅ BGE-M3 모델 초기화 완료: 45000ms (45.0초)
```

**무한 대기 중:**
```
⏳ BGE-M3 모델 초기화 진행 중... (경과: 10.1초, 하트비트: 2회)
(이후 아무 로그도 없음 - 5분 이상)
```

### 방법 3: Supabase `processing_jobs` 테이블 확인

```sql
SELECT 
  id,
  status,
  result->>'status' as rag_status,
  result->>'message' as message,
  result->>'ragElapsed' as rag_elapsed,
  started_at,
  finished_at,
  NOW() - started_at as elapsed_time,
  updated_at,
  NOW() - updated_at as last_update_ago
FROM processing_jobs
WHERE id = 'c2042f81-f724-442e-a960-963607278f4a';
```

**확인 사항:**

**정상 진행 중:**
- `status`: `processing`
- `result->>'status'`: `main_page_rag_processing`
- `updated_at`: 최근 30초 이내 업데이트됨
- `result->>'ragElapsed'`: 계속 증가

**무한 대기 중:**
- `status`: `processing`
- `updated_at`: 30초 이상 업데이트되지 않음
- `result->>'ragElapsed'`: 더 이상 증가하지 않음
