# BGE-M3 모델 서빙 솔루션 비교 및 비용 분석

## 📊 현재 상황 분석

### 사용량 추정
- **크롤링 1회**: 약 200개 청크 (메인 13개 + 하위 페이지 187개)
- **평균 청크 크기**: 200자 (약 50 tokens)
- **월 크롤링 횟수**: 약 10-20회 (초기 데이터 구축 후 감소 예상)
- **월 총 청크 수**: 약 2,000-4,000개 청크
- **월 총 토큰 수**: 약 100,000-200,000 tokens

### 현재 문제점
1. **서버리스 환경 제약**: BGE-M3 초기화 40-90초 (콜드 스타트)
2. **메모리 제한**: Vercel 함수 메모리 제한 (최대 3GB)
3. **타임아웃 위험**: Vercel Pro 최대 5분 실행 시간
4. **캐시 불안정**: `/tmp` 디렉토리는 요청 간 유지되지 않을 수 있음

## 🚀 구현 로드맵

### Phase 1: Google Cloud Run 배포 (1-2일)

1. **FastAPI 서버 구축**
   ```python
   # embedding_api.py
   from fastapi import FastAPI
   from transformers import pipeline
   
   app = FastAPI()
   model = pipeline('feature-extraction', 'Xenova/bge-m3')
   
   @app.post("/embed")
   async def embed(texts: List[str]):
       embeddings = model(texts)
       return {"embeddings": embeddings}
   ```

2. **Docker 컨테이너 생성**
   ```dockerfile
   FROM python:3.11-slim
   COPY . /app
   RUN pip install fastapi uvicorn transformers
   CMD ["uvicorn", "embedding_api:app", "--host", "0.0.0.0", "--port", "8080"]
   ```

3. **Cloud Run 배포**
   ```bash
   gcloud run deploy bge-m3-api \
     --source . \
     --region asia-northeast3 \
     --memory 4Gi \
     --cpu 2 \
     --min-instances 0 \
     --max-instances 10
   ```

4. **Vercel 함수 수정**
   ```typescript
   const response = await fetch('https://bge-m3-api-xxx.run.app/embed', {
     method: 'POST',
     body: JSON.stringify({ texts: chunkTexts }),
   });
   ```

**예상 비용:**
- 월 $7-15
- 초기 설정 시간: 2-4시간
