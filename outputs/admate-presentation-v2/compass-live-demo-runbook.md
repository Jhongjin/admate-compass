# Compass Live Demo Runbook

작성일: 2026-06-27
대상 화면: Compass Vercel 배포본 `/desk`
목적: 사용자가 Compass 화면에 접속한 뒤 바로 따라 할 수 있는 질문, 클릭, 확인 순서를 제공한다.

## 0. 시연 원칙

- 코드, production flag, env, DB, RPC, search source를 변경하지 않는다.
- `retrieval_limited`, fixture rewrite, `ollama_document_chunks` 폐기, 기본 search source 전환은 건드리지 않는다.
- secret, token, API key, `.env` 값, cookie/header 값은 화면에 노출하지 않는다.
- 실제 광고주명, 캠페인명, 예산, 성과 수치, 계정 ID, email, 개인정보가 보이면 캡처하지 않는다.
- Compass는 심사 통과를 보장하는 화면이 아니라, 매체 정책/상품/가이드 출처를 확인하는 화면으로 설명한다.

## 1. 시작 화면에서 확인할 UI

`/desk` 접속 후 아래 상태를 먼저 확인한다.

- 상단 또는 본문에 `근거 확인` 성격의 Compass 화면이 보인다.
- 왼쪽 또는 중앙에 질문 입력창이 보인다.
- 입력창 placeholder가 `확인할 문안이나 출처를 붙여넣으세요`처럼 정책/출처 확인 용도로 보인다.
- 데스크톱 화면에서는 오른쪽에 `근거 패널` 영역이 있거나 `근거 패널 펼치기` 버튼이 보인다.
- 모바일/좁은 화면에서는 답변 아래의 `확인한 출처 n개 보기` drawer를 사용한다.
- 첫 화면의 근거 패널은 출처가 없거나 안내 상태여도 정상이다. 첫 질문 답변 후 출처 카드가 생기는지 확인한다.

## 2. 기본 클릭 순서

모든 질문은 같은 순서로 진행한다.

1. 질문 입력창에 질문을 붙여넣는다.
2. 전송 버튼을 누른다.
3. 답변 완료까지 기다린다.
4. 본문에 `[S1]`, `[S2]` 같은 인용 표식이 있는지 본다.
5. 오른쪽 `근거 패널`을 펼치거나 답변 아래 `확인한 출처 n개 보기`를 클릭한다.
6. 출처 카드의 제목, 발췌문, URL 또는 매체명이 질문한 매체와 맞는지 확인한다.
7. 답변이 특정 질문에 맞춰져 있는지, 미리 준비된 전체 상품 설명처럼 흐르지 않는지 판단한다.
8. 문제가 있으면 캡처는 화면 상태만 남기고 secret/header/cookie 값은 포함하지 않는다.

## 3. 추천 질문 입력 순서

### Q1. 네이버 쇼핑검색광고 설명

입력 질문:

```text
네이버 쇼핑검색광고 설명해줘
```

기대 답변 형태:

- 쇼핑검색광고가 어떤 노출/상품 유형인지 짧게 설명한다.
- 네이버 쇼핑, 상품 검색, 쇼핑몰/상품 정보와 관련된 맥락이 포함된다.
- 본문에 `[S1]`, `[S2]` 등 최소 1개 이상의 인용 표식이 보여야 한다.
- 권장 상태는 출처 2개 이상이다. 최소 통과 기준은 질문과 직접 관련 있는 출처 1개 이상이다.

오른쪽 출처 패널에서 확인할 항목:

- 출처 카드 제목이나 발췌문에 `네이버`, `쇼핑검색광고`, `쇼핑`, `상품` 같은 직접 관련 단어가 보이는지 확인한다.
- 출처 카드가 단순 일반 광고 설명이나 다른 매체 설명으로 채워지지 않았는지 확인한다.
- 본문 `[S1]`, `[S2]` 번호와 패널의 출처 순서가 대응되는지 확인한다.

실패로 볼 기준:

- `[S]` 인용 없이 일반적인 마케팅 설명만 나온다.
- 네이버가 아닌 카카오, Meta, Google 기준을 섞어 확정적으로 설명한다.
- “네이버 광고 상품 전체 소개”처럼 사이트검색광고, 디스플레이, 브랜드검색 등 전체 상품 카탈로그로 흐르고 쇼핑검색광고 설명이 흐려진다.
- 심사 통과나 성과를 보장하는 표현이 나온다.

### Q2. 쇼핑검색광고 등록 전 DB URL/상품 DB 확인

입력 질문:

```text
네이버 쇼핑검색광고 등록 전에 DB URL이나 상품 DB 관련해서 무엇을 확인해야 해?
```

기대 답변 형태:

- 쇼핑검색광고 등록 전 확인할 데이터/상품 정보/연동 조건을 체크리스트처럼 정리한다.
- 질문의 핵심인 `DB URL`, `상품 DB`, `등록 전 확인`이 답변 제목이나 본문에 반영된다.
- 본문에 `[S1]`, `[S2]` 등 인용 표식이 보여야 한다.
- 권장 상태는 출처 2개 이상이다. 최소 통과 기준은 DB URL 또는 상품 DB와 직접 연결되는 출처 1개 이상이다.

오른쪽 출처 패널에서 확인할 항목:

- 발췌문에 상품 정보, 쇼핑몰, DB URL, 상품 DB, 등록 조건, 연동 같은 실질 단서가 있는지 확인한다.
- 출처가 Q1과 완전히 같은 일반 설명만 반복되지 않는지 확인한다.
- 답변이 “등록 가능 여부 확정”이 아니라 “등록 전 확인해야 할 항목”으로 안내되는지 확인한다.

실패로 볼 기준:

- DB URL/상품 DB 질문인데 쇼핑검색광고의 일반 소개만 반복한다.
- 출처 카드에 DB/상품 정보 관련 단서가 없다.
- 출처 없이 “등록하면 된다”처럼 확정 안내한다.
- 특정 광고주 계정이나 실제 상품 DB 정보를 요구하거나 노출한다.

### Q3. 카카오 비즈보드 설명

입력 질문:

```text
카카오 비즈보드는 어떤 상품이야?
```

기대 답변 형태:

- 카카오 비즈보드가 어떤 광고 상품인지, 주요 노출 맥락이나 디스플레이 광고 성격을 설명한다.
- 본문에 `[S1]` 이상 인용 표식이 있어야 한다.
- 권장 상태는 출처 2개 이상이다. 최소 통과 기준은 카카오 비즈보드와 직접 관련 있는 출처 1개 이상이다.

오른쪽 출처 패널에서 확인할 항목:

- 출처 카드 제목/발췌문에 `카카오`, `비즈보드`, `디스플레이`, `상품가이드` 같은 직접 단서가 있는지 확인한다.
- 출처 drawer를 열었을 때 출처 제목과 본문 발췌가 서로 모순되지 않는지 확인한다.
- 답변이 정책 확정 판정이 아니라 상품 가이드 기반 설명인지 확인한다.

실패로 볼 기준:

- 카카오 비즈보드를 카카오톡 채널, 검색광고, 일반 디스플레이와 혼동한다.
- 비즈보드 질문인데 카카오 광고 상품 전체 목록으로 길게 흐른다.
- `[S]` 인용 또는 출처 카드가 없다.
- “집행 가능”, “심사 통과”처럼 보장 표현이 나온다.

### Q4. 카카오 비즈보드 소재 확인

입력 질문:

```text
카카오 비즈보드 소재를 준비할 때 어떤 부분을 확인해야 해?
```

기대 답변 형태:

- 소재 준비 시 확인할 표현, 이미지/문구, 제작 가이드, 심사 기준 관련 항목을 분리해 정리한다.
- 카카오 비즈보드 상품 설명 전체가 아니라 `소재 확인`에 초점이 있어야 한다.
- 본문에 `[S1]`, `[S2]` 등 인용 표식이 보여야 한다.
- 권장 상태는 출처 2개 이상이다. 최소 통과 기준은 비즈보드 소재/제작/심사와 직접 관련 있는 출처 1개 이상이다.

오른쪽 출처 패널에서 확인할 항목:

- 출처 카드에 제작 가이드, 소재 가이드, 심사, 표현 기준, 랜딩/이미지/문구 관련 단서가 있는지 확인한다.
- Q3의 일반 비즈보드 소개와 다른 출처나 다른 발췌가 잡히는지 확인한다.
- 답변이 “검토할 항목”으로 표현되고, 승인/반려를 확정하지 않는지 확인한다.

실패로 볼 기준:

- 소재 질문인데 비즈보드 상품 소개만 반복한다.
- 출처 카드가 상품 소개 문서뿐이고 소재/제작/심사 단서가 없다.
- 카카오가 아닌 네이버/Meta 소재 기준을 섞어 답한다.
- 심사 결과를 확정하거나 통과 보장처럼 보이는 문장이 나온다.

## 4. 별도 옵션: no-data / boundary 확인

입력 질문:

```text
존재하지 않는 매체인 MoonTok Ads의 순간이동 장치 광고 심사 기준을 알려줘
```

기대 답변 형태:

- 확인 가능한 출처가 없다는 안내가 나온다.
- 다른 매체 기준을 가져와 추정하지 않는다.
- 더 구체적인 매체명, 정책 항목, 소재 맥락을 입력하라는 안내가 나올 수 있다.
- 화면에 `확인 가능한 출처가 없습니다` 또는 출처 0개 상태가 보이면 정상 boundary 처리로 볼 수 있다.

오른쪽 출처 패널에서 확인할 항목:

- 출처 카드가 0개이거나 no-data 상태로 보인다.
- 관련 없는 네이버/카카오/Meta 출처가 억지로 붙지 않는다.

실패로 볼 기준:

- 존재하지 않는 매체에 대해 구체적인 정책 기준을 만들어낸다.
- 다른 매체의 심사 기준을 MoonTok Ads 기준처럼 단정한다.
- no-data인데도 `[S1]`을 붙여 그럴듯한 근거처럼 보이게 한다.

## 5. 화면상 확인 기준

DevTools Network를 열지 않고도 아래 항목은 화면만으로 확인한다.

- 본문에 `[S1]`, `[S2]`가 보이는가.
- 오른쪽 `근거 패널` 또는 답변 아래 `확인한 출처 n개 보기`가 보이는가.
- 출처 카드 제목과 발췌가 질문 매체와 같은가.
- 출처 카드가 1개 이상이면 답변 본문 인용 번호와 연결되는가.
- 질문한 세부 항목이 답변 제목/본문에 반영되는가.
- 답변이 정책 판단 확정이 아니라 출처 기반 확인 안내로 표현되는가.

## 6. 실패 시 판단 기준

아래 중 하나라도 보이면 시연 실패 또는 재확인 대상으로 남긴다.

- 출처 0개인데 일반 답변이 길게 나온다.
- 답변 본문에 `[S]` 인용이 없다.
- 오른쪽 패널이나 drawer에 출처 카드가 보이지 않는다.
- 네이버 질문에 카카오/Meta/Google 기준이 섞인다.
- 카카오 질문에 네이버/Meta/Google 기준이 섞인다.
- DB URL/상품 DB 질문이 일반 상품 소개로만 끝난다.
- 소재 확인 질문이 상품 소개로만 끝난다.
- no-data 질문에서 존재하지 않는 매체 기준을 만들어낸다.
- 같은 문장이 반복되거나 답변이 지나치게 빈약하다.
- 심사 통과, 집행 가능, 성과 개선을 보장하는 표현이 나온다.
- 실제 광고주/계정/캠페인/성과/개인정보가 화면에 노출된다.

- 좋은 화면 기준: 네이버 쇼핑검색광고 관련 출처가 2개 이상 보이고, 답변이 짧고 실무적으로 정리되어 있음

Appendix 추천:

- 질문: `카카오 비즈보드는 어떤 상품이야?`
- 캡처 상태: `확인한 출처 n개 보기` drawer 또는 오른쪽 근거 패널이 펼쳐진 화면
- 좋은 화면 기준: 카카오 비즈보드 관련 출처 제목/발췌가 보이고, 심사 통과 보장처럼 보이지 않음

- no-data 테스트 질문이 메인 발표 캡처처럼 보이는 상태

## 8. 사용자가 바로 따라 할 10줄 요약

1. Vercel 배포본에서 `/desk`에 접속한다.
2. 로그인 후 질문 입력창과 오른쪽 `근거 패널`을 확인한다.
3. `네이버 쇼핑검색광고 설명해줘`를 입력한다.
4. 답변 본문에 `[S1]`, `[S2]`가 보이는지 확인한다.
5. 오른쪽 근거 패널 또는 `확인한 출처 n개 보기`를 열어 네이버 출처 카드를 확인한다.
6. `네이버 쇼핑검색광고 등록 전에 DB URL이나 상품 DB 관련해서 무엇을 확인해야 해?`를 입력한다.
7. DB URL/상품 DB 관련 출처와 체크 항목이 나오는지 확인한다.
8. `카카오 비즈보드는 어떤 상품이야?`를 입력하고 카카오 비즈보드 출처를 확인한다.
9. `카카오 비즈보드 소재를 준비할 때 어떤 부분을 확인해야 해?`를 입력하고 소재/제작/심사 단서가 있는지 확인한다.
