# 청크 실패 문서 진단 가이드

## 실패 원인 확인 방법

### 1. 브라우저에서 확인
실패한 문서를 클릭하여 상세 정보 확인

### 2. API로 확인
```bash
# 실패한 문서 ID로 청크 정보 확인
GET /api/admin/check-document-chunks?documentId=YOUR_DOCUMENT_ID
```

### 3. 터미널 로그 확인 포인트

다음 로그들을 확인하세요:

#### A. 문서 내용 확인
```
❌ PDF 바이너리 데이터 감지 - 텍스트 추출 실패로 인해 청킹 불가
```
→ PDF에서 텍스트를 추출하지 못한 경우

#### B. 청킹 결과 확인
```
[CRITICAL] 📊 최종 청킹 결과 (반드시 출력):
  chunkCount: 0
  note: "⚠️ 1개 청크만 생성됨" 또는 "⚠️ 청크가 없음"
```
→ 청킹이 실패하거나 0개 청크가 생성된 경우

#### C. 문서 구조 분석 확인
```
[CRITICAL] 📊 문서 구조 분석 결과:
  sectionsCount: 1
  paragraphsCount: 1
```
→ 문서 구조 분석이 실패한 경우 (이미 개선됨)

#### D. 통합 청킹 실패 확인
```
❌ 통합 청킹 결과가 비어있습니다. 상세 정보:
  contentLength: ...
  contentPreview: ...
```
→ 통합 청킹 서비스가 청크를 생성하지 못한 경우

#### E. 문서 내용이 비어있음
```
❌ 문서 내용이 비어있습니다. 상세 정보:
  originalContentLength: ...
  cleanedContentLength: 0
```
→ 텍스트 인코딩 후 내용이 비어있는 경우

### 4. 일반적인 실패 원인

1. **PDF 텍스트 추출 실패**
   - 이미지 기반 PDF
   - 손상된 PDF 파일
   - 텍스트 레이어가 없는 PDF

2. **문서 내용이 비어있음**
   - 인코딩 문제
   - 빈 파일
   - 텍스트 추출 실패

3. **청킹 로직 실패**
   - 문서 구조 분석 실패
   - 통합 청킹 서비스 오류
   - 폴백 청킹도 실패

4. **임베딩 생성 실패**
   - 모델 초기화 실패
   - 타임아웃
   - 메모리 부족

### 5. 해결 방법

1. **재인덱싱 시도**
   - 문서 상세 페이지에서 "재인덱싱" 버튼 클릭
   - 또는 API 호출: `POST /api/admin/upload/[documentId]/reindex`

2. **파일 확인**
   - PDF가 텍스트를 포함하는지 확인
   - 파일이 손상되지 않았는지 확인

3. **로그 확인**
   - 터미널에서 위의 로그 메시지 확인
   - 브라우저 콘솔에서 에러 확인
