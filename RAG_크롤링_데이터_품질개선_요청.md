# RAG 챗봇 크롤링 데이터 품질 개선 요청

---

## 1. 현재 문제 상황

Supabase `documents` 테이블의 크롤링 결과를 확인한 결과, 다음 두 가지 데이터 품질 이슈가 확인되었습니다.

### 1-1. title 중복 문제

- 대부분의 row에서 title이 **"상품 더 알아보기"**, **"자세히 알아보기"** 등으로 반복 수집
- 실제 페이지 제목이 아닌 **버튼/링크 텍스트**가 title로 잘못 캡처된 것으로 판단

### 1-2. content 중복 문제

- 거의 모든 row의 content가 아래 패턴으로 동일:
  ```
  카테고리 단가도움말 카테고리카테고리 단검색어 입력 창텍스트 자주기 광고 시작 회원/계정관리 메뉴 펼치기 검색광고 광고 상품메뉴 url
  ```
- 페이지 본문이 아닌 **네비게이션, 사이드바, 헤더 등 공통 레이아웃 요소**가 본문과 함께 수집된 상태
- 이로 인해 **RAG 검색 시 관련 없는 chunk가 반환**되어 답변 품질이 저하됨

## 2. 개선 요청 사항

### 2-1. 크롤러 본문 추출 로직 수정 (최우선)

| 항목 | 현재 | 개선 방향 |
|------|------|-----------|
| **본문 추출** | 페이지 전체 HTML 텍스트화 | `<main>`, `<article>` 등 본문 컨테이너만 선택적 추출 |
| **레이아웃 요소** | 함께 수집됨 | `<nav>`, `<header>`, `<footer>`, 사이드바 영역 명시적 제외 |
| **title 추출** | 버튼/링크 텍스트 캡처 | `<h1>` 또는 `<title>` 태그에서 실제 페이지 제목 추출 |

**구체적 작업 내용:**

1. 크롤링 대상 사이트의 HTML 구조를 분석하여 **본문 영역의 CSS selector 또는 XPath를 특정**
2. `<nav>`, `<header>`, `<footer>`, 사이드바, 공통 메뉴 영역을 **크롤링 대상에서 제외**
3. title은 `<h1>` → `<title>` → `og:title` 순으로 **우선순위 기반 추출** 적용
4. JavaScript 렌더링이 필요한 페이지의 경우, 본문 로딩 완료 후 추출되도록 **대기 로직 확인**

### 2-2. 데이터 후처리 로직 추가

1. **보일러플레이트 제거**
   - 전체 문서에서 공통으로 등장하는 텍스트 패턴을 식별하여 자동 제거
   - 방법: TF-IDF 기반 문서 빈도(DF)가 일정 비율 이상인 문구 필터링

2. **중복 content 제거**
   - content 컬럼 기준 코사인 유사도 90% 이상인 row는 하나만 유지
   - 또는 content 해시값 기반 완전 중복 제거

3. **최소 품질 기준 적용**
   - 보일러플레이트 제거 후 본문 길이가 일정 기준(예: 50자) 미만인 문서는 제외
   - title이 비어있거나 공통 UI 텍스트인 경우 플래그 처리

### 2-3. 기존 데이터 정제

- 현재 `documents` 테이블에 적재된 데이터에 대해 **일괄 정제 스크립트 실행** 필요
- 정제 후 관련 **임베딩(벡터) 데이터도 재생성** 필요
