# Meta FAQ AI 챗봇 프로젝트 분석 보고서

## 📋 프로젝트 기능

### 핵심 기능
- **RAG 기반 AI 챗봇**: Google Gemini를 활용한 한국어 질문-답변 시스템
- **문서 업로드/인덱싱**: PDF, DOCX, TXT 파일 및 URL 크롤링 지원
- **벡터 검색**: pgvector를 활용한 의미 기반 문서 검색
- **실시간 채팅**: 스트리밍 답변 및 출처 표시
- **히스토리 관리**: 개인별 대화 기록 및 즐겨찾기 기능
- **피드백 시스템**: 답변 품질 평가 (👍/👎/⭐)
- **관리자 대시보드**: 시스템 모니터링 및 문서 관리
- **이메일 알림**: 사용자 문의 및 시스템 오류 알림
- **문서 관리**: 업로드, 다운로드, 삭제, 상태 모니터링

### 고급 기능
- **URL 크롤링**: Puppeteer를 활용한 웹페이지 자동 수집
- **사이트맵 발견**: 자동 사이트맵 탐지 및 크롤링
- **텍스트 인코딩**: UTF-8 기반 다국어 텍스트 처리
- **배치 처리**: 대용량 문서의 청크 단위 처리
- **중복 방지**: 해시 기반 문서 중복 검사
- **로깅 시스템**: 상세한 처리 로그 및 오류 추적

## 🛠️ 적용된 기술(플랫폼)

### 프론트엔드
- **Next.js 15**: App Router 기반 풀스택 프레임워크
- **TypeScript**: 정적 타입 검사 및 코드 안정성
- **React**: 컴포넌트 기반 UI 개발
- **Tailwind CSS**: 유틸리티 기반 스타일링
- **shadcn/ui**: 재사용 가능한 UI 컴포넌트
- **Framer Motion**: 애니메이션 및 인터랙션
- **Lucide React**: 아이콘 시스템

### 백엔드 & 데이터베이스
- **Supabase**: PostgreSQL + pgvector 벡터 데이터베이스
- **Google Gemini**: LLM (gemini-2.5-flash-lite)
- **Vercel**: 서버리스 배포 및 호스팅
- **LangChain**: RAG 파이프라인 구축
- **Puppeteer**: 웹 크롤링 및 스크린샷

### 데이터 처리
- **Xenova/Transformers**: BGE-M3 임베딩 모델
- **PDF-Parse**: PDF 텍스트 추출
- **Mammoth**: DOCX 텍스트 추출
- **Tesseract**: OCR 텍스트 인식
- **Whisper**: 음성-텍스트 변환

### 개발 도구
- **Zod**: 스키마 검증
- **Zustand**: 상태 관리
- **React Query**: 서버 상태 관리
- **React Hook Form**: 폼 관리
- **Date-fns**: 날짜 처리
- **ES-Toolkit**: 유틸리티 함수

## ⚠️ 보완사항

### 기술적 제약사항
- **PDF/DOCX 텍스트 추출 제한**: 서버리스 환경에서 실제 텍스트 추출 불가
- **서버리스 환경 제약**: Vercel의 30초 타임아웃으로 대용량 파일 처리 한계
- **이메일 서비스 비활성화**: 실제 이메일 발송 기능 미구현
- **인증 시스템 미완성**: Supabase Auth 연동 필요
- **에러 처리 개선**: 더 세밀한 오류 분류 및 사용자 피드백 필요

### 성능 최적화
- **임베딩 모델 최적화**: BGE-M3 모델 로딩 시간 단축
- **벡터 검색 성능**: 인덱스 최적화 및 쿼리 성능 개선
- **캐싱 전략**: 자주 사용되는 데이터 캐싱 구현
- **배치 처리**: 대용량 문서 처리 최적화

### 보안 강화
- **파일 업로드 보안**: 악성 파일 검사 및 크기 제한
- **API 보안**: Rate limiting 및 인증 강화
- **데이터 암호화**: 민감한 데이터 암호화 저장
- **접근 권한**: 세분화된 사용자 권한 관리
